# idea炉 · 使用指南

> 基于《文案的基本修养》Idea 金字塔方法论的传播策划 skill。
> 把洞察和策略熔炼成完整传播方案。

---

## 这是什么

`idea炉` 是一个 Claude Code skill，帮你用**Idea 金字塔**方法论做传播策划。

适用场景：内容策划、选题策划、品牌传播、活动策划、小红书/公众号推广方案、公益倡导等。

核心逻辑：
```
商业目标 → 目标人群心象 → 核心体验 → 传播目标(A→B)
    → 洞察 → 策略Idea → 创意Idea → 执行Idea
```

---

## 安装方法

将 `idea炉/` 文件夹放入你的 Claude Code skills 目录：

```bash
# 全局安装（所有项目可用）
cp -r idea炉 ~/.claude/skills/

# 或项目级安装
cp -r idea炉 .claude/skills/
```

重启 Claude Code 后，输入 `/idea炉` 即可调用。

## 四种使用模式

### 模式 A｜直接产出
信息充足时，直接给出完整策划方案。

```
/idea炉 我有一个教年轻人记账的公众号，目标人群是月薪5k左右的应届生，
希望他们下载 App 开始记账，主要在小红书传播
```

---

### 模式 B｜先问后做
信息模糊时，Claude 会先问 2-3 个关键问题再产出。

```
/idea炉 帮我做一个咖啡品牌的传播方案
```
→ Claude 会问：品牌的核心体验是什么？主要面向哪类人群？

---

### 模式 C｜快速脑暴
只需要快速思路，不要完整方案。

```
/idea炉 随便给我几个方向，关于职场焦虑的内容选题
```
→ 直接输出 3 个不同角度的创意，每个附一句执行建议。

---

### 模式 D｜双向费曼共创 ⭐ 推荐
不是 AI 帮你想，而是你先说、AI 来挑战——最终碰撞出比任何一方单独能想到的都更成熟的方案。
适合：你对自己的产品/用户有深入了解，想通过对话把模糊的直觉变成清晰策略。

**你已有策划思路时：**
```
/idea炉 引导我思考 [你的项目简介 + 你的策划思路]
```
→ Claude 先读你的思路，指出最多 3 个疑问，再问 5 个追问（从基础到刁钻），最后把你的直觉和 Idea 金字塔方法论碰撞出完整方案。

**你还没有策划思路时：**
```
/idea炉 引导我思考 [你的项目简介]
```
→ Claude 先问你：「你先试着解释一下——这个项目的核心价值是什么？你的策划直觉是什么方向？」
→ 你解释后，进入费曼检验流程。

> 💡 **提示**：这个模式下，Claude 会化身严格的「创意总监」对你发出连环追问。别怕答错，哪怕你回答「我不知道」，这种碰撞也能帮你排雷。

流程：你解释 → AI 批评（≤3点）→ AI 追问（5问）→ 双向碰撞 → 完整方案（标注人机各自贡献）

## 方法论速查

### 三种洞察

| 类型 | 用途 | 检验标准 |
|------|------|----------|
| 消费者洞察 | 触动认知，产生创意Idea | 「太对了！就是这么回事！」 |
| 参与者洞察 | 引发互动，扩大传播 | 受众会主动转发/参与 |
| 社会群体洞察 | 破圈，吸引更广泛关注 | 非目标用户也有共鸣 |

### Idea 三层级

| 层级 | 回答的问题 | 决定因素 |
|------|-----------|----------|
| 策略 Idea | 需要实现怎样的认知改变？ | 市场判断 + 产品理解 |
| 创意 Idea | 让他们感知到什么才能改变？ | 洞察 × 策略 |
| 执行 Idea | 做怎样的内容来传达？ | 创作才华 |

### 传播目标格式

**商业类：**
> 通过让 [某些人] [做出某种行为]，实现 [商业结果]

**公益/倡导类：**
> 通过让 [某些人] [改变对某件事的看法或行为]，实现 [社会影响目标]

## 使用技巧

**给的背景越具体，方案越有用**
不只是说「帮我做传播方案」，而是告诉 Claude：
- 产品/内容是什么，它的核心差异点在哪
- 目标人群是谁，他们现在对这件事什么态度
- 希望在哪个平台传播
- 有没有资源限制（预算、人手、时间）

**推断型内容会被标注**
Claude 基于推断（而非你提供的真实数据）产出的洞察/分析，会注明「推断型，建议验证」。这是正常现象，说明你需要用一手调研来补充或验证。

**多任务需求按序产出**
如果你的 brief 包含「调研分析 + 传播策略」，Claude 会先完成分析部分，再进入策划流程，不会跳步。
